# 追加機能計画書：履歴機能

## 1. 概要

文字起こしを行った過去の記録（日時、内容）を保存し、アプリケーション上で閲覧・再確認できる「履歴機能」を追加する。

## 2. 目的

- ユーザーが過去の文字起こし結果を振り返ることができるようにする。
- アプリケーションを再起動しても、以前の作業内容が失われないようにする。

## 3. 機能詳細

### 3.1 データ保存

- 文字起こし完了時に、以下の情報をローカルファイル（`history.json`）に保存する。
  - 日時 (timestamp)
  - 文字起こしテキスト (text)
  - 保存されたファイルパス (file_path)

### 3.2 UI 変更

- メインウィンドウを拡張し、右側に「履歴リスト」を表示するエリアを追加する。
- 履歴リストには「日時」を表示する。

### 3.3 ユーザー操作

- 履歴リストの項目をクリックすると、その履歴の「文字起こしテキスト」が結果表示エリアに表示される。
- アプリ起動時に、保存された履歴データを読み込んでリストを表示する。

## 4. 実装予定ファイル

- `main.py`: UI の変更、履歴データの読み書きロジックの追加。
- `history_manager.py` (新規): 履歴データの管理（保存・読み込み）を担当するクラス。
